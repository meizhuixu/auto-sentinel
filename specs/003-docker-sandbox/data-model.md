# Data Model: Sprint 3 - Secure Docker Sandbox Execution

## New Entity: ExecutionResult

Represents the outcome of running the `fix_script` inside the Docker sandbox. Stored in `DiagnosticState["execution_result"]`.

| Field | Type | Description |
|-------|------|-------------|
| `status` | `str` | One of: `"success"`, `"failure"`, `"timeout"`, `"error"`, `"skipped"` |
| `return_code` | `Optional[int]` | Container exit code; `None` for timeout/error/skipped |
| `stdout` | `str` | Captured standard output from the container |
| `stderr` | `str` | Captured standard error from the container |
| `duration_ms` | `int` | Wall-clock time from container start to completion (or timeout), in milliseconds |
| `error` | `Optional[str]` | Docker-level error message; populated on `"error"` or `"timeout"` status |

**Status values**:
| `status` | Meaning |
|----------|---------|
| `success` | Script exited with return code 0 |
| `failure` | Script exited with non-zero return code |
| `timeout` | Script exceeded the 5-second timeout; container was killed |
| `error` | Docker-level error (daemon unavailable, container failed to start, etc.) |
| `skipped` | `fix_script` was `None`; no container was started |

## format_report: New Sandbox Execution Section

Appended to the existing report template after "Remediation Steps":

```markdown
## Sandbox Execution

**Status**: {status}
**Return code**: {return_code}
**Duration**: {duration_ms}ms

### Output

```
{stdout}
```

### Errors

```
{stderr}
```
```

When `execution_error` is set instead of `execution_result`:

```markdown
## Sandbox Execution

**Status**: error
**Reason**: {execution_error}
```

When `execution_result.status == "skipped"`:

```markdown
## Sandbox Execution

**Status**: skipped (no fix script generated)
```
