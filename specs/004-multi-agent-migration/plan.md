# Implementation Plan: Sprint 4 - Multi-Agent Migration

**Branch**: `004-multi-agent-migration` | **Date**: 2026-04-28 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/004-multi-agent-migration/spec.md`

## Summary

Evolve AutoSentinel from a linear 4-node LangGraph pipeline into a 6-agent
multi-agent architecture (Supervisor + Diagnosis + CodeFixer + InfraSRE +
SecurityReviewer + Verifier). All agents are mock-only in Sprint 4 (`# TODO(W2)`).
The FastAPI gateway, asyncio.Queue, and `run_pipeline()` interface are unchanged.
A feature flag (`AUTOSENTINEL_MULTI_AGENT=1`) switches between v1 and v2 graphs,
enabling a 5-scenario smoke benchmark. LangGraph 1.1.9 fan-out (with
`Annotated[list, operator.add]` reducers) and `interrupt()+Command(resume=)` are
the confirmed parallel/interrupt mechanisms.

## Key Technical Decisions

### 1. Fan-Out Requires `Annotated` Reducers (Confirmed by Test)

Naive fan-out with plain TypedDict fields raises `InvalidUpdateError`. Only `agent_trace` (written by multiple parallel branches) needs the reducer. `fix_artifact` and `security_verdict` are exclusive per branch — no reducer needed.

### 2. `interrupt()` Requires a Checkpointer

`build_multi_agent_graph()` MUST compile with `checkpointer=MemorySaver()`. Tests pass a `{"configurable": {"thread_id": ...}}` config. Production upgrade to a persistent checkpointer (e.g., PostgreSQL) is Sprint 5 scope.

### 3. Security Reviewer Reads `fix_script` in Parallel Step

In the fan-out, both parallel nodes see the same pre-step state. SecurityReviewer cannot see CodeFixer's `fix_artifact` during the parallel step. In Sprint 4 mock mode, it runs keyword check on `state["fix_script"]` (Sprint 3 field set by `analyze_error`). This is documented as a Sprint 5 TODO.

### 4. CI Docker Import Check — stdlib `ast`, No New Dependency

`test_docker_import_boundary.py` uses `ast.walk()` to find `import docker` / `from docker` in all `.py` files under `autosentinel/`, asserts only `autosentinel/agents/verifier.py` is in the allowlist. Runs in CI as part of the pytest suite.

### 5. Feature Flag for v1/v2 Benchmark

`AUTOSENTINEL_MULTI_AGENT=1` env var switches `build_graph()` call in `run_pipeline()`. Benchmark module calls `run_pipeline()` twice per scenario with different env vars. No interface changes to the FastAPI gateway or asyncio.Queue.

### 6. `caution_flag` Removed — `security_verdict` Sufficient

`format_report` reads `state.get("security_verdict")` directly. A `CAUTION` verdict renders a `⚠ CAUTION` badge. No separate boolean field in AgentState.
