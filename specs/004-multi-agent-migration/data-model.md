# Data Model: Sprint 4 - Multi-Agent Migration

## New Entity: AgentState

Extends `DiagnosticState` with six new fields. All Sprint 1–3 fields are preserved
unchanged. `AgentState` is the single TypedDict used by every node in the v2 graph.

### Full Field Table

| Field | Type | Set by | Sprint |
|-------|------|--------|--------|
| `log_path` | `str` | caller | 1 |
| `error_log` | `Optional[ErrorLog]` | `parse_log` | 1 |
| `parse_error` | `Optional[str]` | `parse_log` | 1 |
| `analysis_result` | `Optional[AnalysisResult]` | `analyze_error` (v1 only) | 1 |
| `analysis_error` | `Optional[str]` | `analyze_error` (v1 only) | 1 |
| `fix_script` | `Optional[str]` | `analyze_error` (v1 only) | 3 |
| `execution_result` | `Optional[ExecutionResult]` | `execute_fix` (v1) / `VerifierAgent` (v2) | 3 |
| `execution_error` | `Optional[str]` | `execute_fix` (v1) / `VerifierAgent` (v2) | 3 |
| `report_text` | `Optional[str]` | `format_report` | 1 |
| `report_path` | `Optional[str]` | `format_report` | 1 |
| `error_category` | `Optional[str]` | `DiagnosisAgent` | **4 NEW** |
| `fix_artifact` | `Optional[str]` | `CodeFixerAgent` / `InfraSREAgent` | **4 NEW** |
| `security_verdict` | `Optional[str]` | `SecurityReviewerAgent` | **4 NEW** |
| `routing_decision` | `Optional[str]` | `SupervisorAgent` | **4 NEW** |
| `agent_trace` | `Annotated[list[str], operator.add]` | all agents (parallel-safe) | **4 NEW** |
| `approval_required` | `bool` | `security_gate` node | **4 NEW** |

### Field Notes

**`error_category`**: One of `CODE`, `INFRA`, `CONFIG`, `SECURITY`. Maps from Sprint 1–3
`AnalysisResult.error_category` values as follows:

| Sprint 1–3 value | Sprint 4 value | Routing target |
|-----------------|---------------|---------------|
| `connectivity` | `INFRA` | `InfraSREAgent` |
| `resource_exhaustion` | `INFRA` | `InfraSREAgent` |
| `configuration` | `CONFIG` | `InfraSREAgent` |
| `application_logic` | `CODE` | `CodeFixerAgent` |
| *(unknown/fallback)* | `CODE` | `CodeFixerAgent` |

> **Consistency note (resolves US1 scenario 3)**: There is no separate `ConfigAgent`.
> `CONFIG` and `INFRA` both route to `InfraSREAgent`. "Config Agent path" in US1
> scenario 3 refers to the CONFIG routing branch, not a distinct agent class.

**`fix_artifact`**: The fix script or patch string generated by the active specialist
agent. Replaces `fix_script` (Sprint 3) in the v2 pipeline. `fix_script` is retained
in AgentState for v1 pipeline compatibility — in the v2 graph `fix_artifact` is what
the VerifierAgent executes.

**`security_verdict`**: One of `SAFE`, `CAUTION`, `HIGH_RISK`. Set by
`SecurityReviewerAgent`. Read by `security_gate` node and `format_report`.

> **Consistency note (resolves caution_flag)**: There is no separate `caution_flag`
> boolean field. `format_report` reads `security_verdict` directly and renders a
> "⚠ CAUTION" badge when `security_verdict == "CAUTION"`. No redundant field needed.

**`routing_decision`**: Human-readable string recording the Supervisor's routing choice
and arbitration rationale. Example: `"CODE → CodeFixerAgent (error_category=CODE)"`.

**`agent_trace`**: List of agent names appended by each agent that ran. Uses
`Annotated[list[str], operator.add]` reducer so parallel branches can both append
without conflict. Example after a CODE run: `["DiagnosisAgent", "CodeFixerAgent",
"SecurityReviewerAgent", "VerifierAgent"]`.

**`approval_required`**: Set to `True` by the `security_gate` node when
`security_verdict == "HIGH_RISK"` and `interrupt()` is about to be issued.
Allows `format_report` to record that the pipeline was suspended for human review.

## New Entity: BaseAgent

Abstract class. All six specialist/router agents inherit from it.

```python
from abc import ABC, abstractmethod

class BaseAgent(ABC):
    @abstractmethod
    def run(self, state: AgentState) -> AgentState: ...
```

The LangGraph node wrapper is a one-liner:

```python
builder.add_node("diagnosis_agent", lambda s: diagnosis_agent.run(s))
```

`run()` receives the full `AgentState`, returns a **partial** dict (not the full state
— LangGraph merges it). Return type is annotated as `AgentState` but in practice only
changed fields need to be returned. All mock implementations MUST include

## Supervisor Routing Table

| `error_category` | Specialist fired | Security Reviewer fired |
|-----------------|-----------------|------------------------|
| `CODE` | `CodeFixerAgent` | Yes (parallel) |
| `INFRA` | `InfraSREAgent` | Yes (parallel) |
| `CONFIG` | `InfraSREAgent` | Yes (parallel) |
| `SECURITY` | `CodeFixerAgent` | Yes (parallel) |
| `UNKNOWN` | `CodeFixerAgent` (fallback) | Yes (parallel) |

Security Reviewer always fires in parallel with the specialist. In the fan-out, both
nodes receive the same state snapshot (pre-fix_artifact). SecurityReviewer in Sprint 4
mock mode runs keyword detection on `state["fix_script"]` (Sprint 3 field, already
set by `analyze_error`) as a proxy for the not-yet-generated `fix_artifact`.

## Security Verdict Classification

### Sprint 4 Mock Rule (keyword-based)

The following keywords in `fix_script` / `fix_artifact` trigger `HIGH_RISK`:

```python
_HIGH_RISK_KEYWORDS = [
    "DROP TABLE", "DROP DATABASE", "TRUNCATE TABLE",
    "rm -rf /", "rm -rf ~", "chmod 777", "mkfs", "dd if=",
]
```

All other fix artifacts → `SAFE` in Sprint 4 mock.

> This keyword list is the **temporary surrogate** defined in Constitution V.
> It MUST be removed and replaced with real LLM classification in Sprint 5.

### Verdict → Pipeline Action

| `security_verdict` | `security_gate` action | Report badge |
|-------------------|----------------------|-------------|
| `SAFE` | Pass through; no interrupt | None |
| `CAUTION` | Pass through; set `approval_required=False` | `⚠ CAUTION` |
| `HIGH_RISK` | Log `human_approval_required`; `interrupt()`; set `approval_required=True` | `🚨 HIGH RISK — awaiting approval` |

## Updated Graph Flow (v2)

```text
START
  │
  ▼
parse_log
  │  parse_error? → END (no report)
  ▼
diagnosis_agent         ← sets: error_category, agent_trace += ["DiagnosisAgent"]
  │
  ▼
supervisor_route        ← sets: routing_decision; conditional fan-out
  │
  ├──► code_fixer_agent     ─────────────────────┐
  │    (if CODE or SECURITY or fallback)          │ parallel fan-out
  │    sets: fix_artifact, agent_trace            │
  │                                               │
  ├──► infra_sre_agent      ─────────────────────┤
  │    (if INFRA or CONFIG)                       │
  │    sets: fix_artifact, agent_trace            │
  │                                               │
  └──► security_reviewer_agent  ─────────────────┘
       (always, in parallel with specialist)
       sets: security_verdict, agent_trace
                                          │
                                          ▼
                                  supervisor_merge
                                  (no-op merge node; state already merged by LangGraph)
                                          │
                                          ▼
                                  security_gate
                                  HIGH_RISK? → log + interrupt() → human approval
                                  SAFE/CAUTION → pass through
                                          │
                                          ▼
                                  verifier_agent      ← sets: execution_result, execution_error
                                  (sole Docker SDK caller)
                                          │
                                          ▼
                                  format_report       ← reads: all fields; appends Sandbox + Security sections
                                          │
                                          ▼
                                        END
```

**v1 graph** (unchanged, used by benchmark):
```text
START → parse_log → analyze_error → execute_fix → format_report → END
```

## format_report: New Security Verdict Section

Appended after the existing "Sandbox Execution" section:

```markdown
## Security Review

**Verdict**: {security_verdict}
**Routing**: {routing_decision}
**Agents**: {", ".join(agent_trace)}
```

When `security_verdict == "CAUTION"`:
```markdown
## Security Review

**Verdict**: ⚠ CAUTION
...
```

When `approval_required == True` (HIGH_RISK, resumed after approval):
```markdown
## Security Review

**Verdict**: 🚨 HIGH RISK — executed after human approval
...
```
